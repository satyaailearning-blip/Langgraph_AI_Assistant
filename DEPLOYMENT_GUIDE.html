1. System Overview & Architecture

1.1 What is This System?

A production-ready AI assistant that:

  • Receives queries via WhatsApp messaging
  • Intelligently routes questions to RAG (document-based) or General (LLM-based) agents
  • Searches PDF documents using semantic embeddings (FAISS)
  • Responds with context-aware answers in 2-5 seconds
  • Runs 24/7 on cloud without user's local machine

1.2 Architecture Diagram

User Query (WhatsApp) ↓ Twilio Webhook ↓ FastAPI Server (Render Cloud) ↓ LangGraph Router ↓ ├→ [Keyword Check] │ └→ Document mentioned? │ ├→ [RAG Agent] │ ├→ Query FAISS Vector DB │ ├→ Retrieve Top 5 Chunks │ └→ Generate Context-Aware Response │ └→ [General Agent] └→ Direct LLM Response (GPT-4o-mini)

5. Twilio Integration Setup

5.1 Why Twilio?

Twilio acts as a bridge between WhatsApp and your FastAPI server. It:

  • Receives WhatsApp messages from users
  • Forwards them to your webhook (/whatsapp endpoint)
  • Converts your responses back to WhatsApp messages
  • Handles rate limiting and message formatting

5.2 Setup Instructions

Part A: Get Twilio Credentials

  1. Visit: https://www.twilio.com/console
  2. Account SID: Copy from dashboard
  3. Auth Token: Copy from dashboard
  4. Note: Keep these secure, never commit to GitHub

Part B: Configure WhatsApp Sandbox

  1. Messaging → Try it Out → Send WhatsApp
  2. Join sandbox with your personal WhatsApp number
  3. Send a test message (e.g., "join example")
  4. Receive confirmation from Twilio

Part C: Update Webhook URL

  1. Still in WhatsApp Console
  2. Find "When a message comes in"
  3. Clear old URL (Cloudflare tunnel)
  4. Enter: https://langgraph-ai-assistant.onrender.com/whatsapp
  5. Method: POST
  6. Save configuration

5.3 Testing the Integration

  1. Open WhatsApp on your phone
  2. Start conversation with your Twilio WhatsApp number
  3. Send message: "Hello"
  4. Expected response: AI assistant responds within 5 seconds

5.4 Webhook Request/Response Format

Incoming Request (Twilio → Your Server):

POST /whatsapp
Content-Type: application/x-www-form-urlencoded

From=whatsapp:+1234567890
To=whatsapp:+2348123456789
Body=What is the adequacy report?

Outgoing Response (Your Server → Twilio):

<Response>
  <Message>Here is the information...</Message>
</Response>
⚠ Important: Render webhook must be HTTPS and publicly accessible. Twilio cannot reach localhost or private IPs.

7.5 Processing Pipeline

PDF Upload
    ↓

    ↓

    ↓
OpenAI Embeddings (Vectorize each chunk)
    ↓
FAISS Index (Create searchable vector DB)
    ↓
Save to /app/Agents/data/vector_db/
    ↓
Ready for Semantic Search

8.2 Adjust Document Processing

File: Agents/rag/document_loader.py

splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,        # Characters per chunk
    chunk_overlap=200       # Character overlap
)
  • chunk_size: Smaller = More chunks, finer granularity
  • chunk_overlap: Prevents losing info at boundaries

8.3 Change LLM Model

File: Agents/graph_builder.py

llm = ChatOpenAI(
    model="gpt-4o-mini",  # Change model here
    temperature=0         # 0=deterministic, 1=creative
)

Available Models:

  • gpt-4o-mini (current - fast & cheap)
  • gpt-4-turbo (more powerful, slower)
  • gpt-3.5-turbo (fastest, less capable)

8.4 Modify Response Constraints

10. Troubleshooting Guide

10.1 Common Issues & Solutions

Issue: "Vector DB not found"

Symptoms: Health endpoint shows "vector_db": "not_ready"

Cause: First deployed or vector DB deleted

Solution:

POST https://langgraph-ai-assistant.onrender.com/rebuild-vector-db

Wait 45-60 seconds for indexing to complete.

Issue: "Document folder not found"

Symptoms: Logs show Error: Document folder not found

Cause: Documents not committed to git

Solution:

  1. Verify PDFs are in Agents/data/documents/
  2. Run: git add Agents/data/documents/
  3. Commit and push
  4. Rebuild vector DB

Issue: "WhatsApp not responding"

Symptoms: Send message but get no response

Causes & Solutions:

Check Action
Render service running? Visit /health endpoint
Twilio webhook correct? Verify URL not Cloudflare tunnel
OpenAI API key valid? Check Render environment vars
Message format? Twilio shows POST errors in logs

Issue: "Slow response times" (>10 seconds)

Causes:

  • OpenAI API rate limits
  • Large number of documents in DB
  • Render free tier overloaded

Solutions:

  • Rebuild vector DB to optimize
  • Upgrade Render plan
  • Reduce chunk size for faster search
  • Check OpenAI status page

Issue: "FAISS index corrupted"

Symptoms: Cryptic FAISS error in logs

Solution: Rebuild vector DB (deletes and recreates index)

10.3 Reset Vector Database

If vector DB becomes corrupted:

  1. SSH into Render or use dashboard
  2. Delete: /app/Agents/data/vector_db/
  3. Call rebuild endpoint
  4. Wait for indexing to complete

10.4 When to Contact Support

If you see these errors, contact support:

  • OPENAI_API_KEY invalid/expired → OpenAI support
  • Twilio webhook authentication failed → Twilio support
  • PostgreSQL connection refused → Render support
  • OutOfMemory exception → Upgrade Render plan

10.5 Emergency Rollback

If deployment breaks:

  1. Find last working commit in GitHub
  2. Run: git revert [broken-commit]
  3. Push to GitHub
  4. Render auto-redeploys to previous version
